# 「你说了算」搜推一体实时意图 Demo

一个适合 3–5 分钟路演的本地 Web Demo：用户可以在淘宝风格首页按住说话，用自然语言表达即时需求，并立刻重排推荐商品。

## 运行

```bash
python3 app.py
```

打开：

```text
http://127.0.0.1:8000
```

推荐使用桌面版 Chrome 或 Edge：首次点击语音按钮时允许麦克风权限，即可使用浏览器自带的真实语音转写。支持“长按说完松开”和“轻点开始、再次轻点结束”两种操作。

Codex 内置浏览器目前不会向本地页面开放麦克风；等待 8 秒后页面会明确切换为“演示识别”，使用当前场景示例口令完成交互。也可以随时输入文字或直接点击示例语句。Demo 不保存录音，也不请求淘宝、外部大模型或图片服务。

## 推荐路演路径

1. 点击首页拟人语音按钮，或在页面空白处按住说话。
2. 连续商品意图：自行依次说“给我推荐一些好看的裙子”“想要韩式风格”“想要颜色浅一点”，观察推荐从多风格裙装切换为深色韩式，再切换为浅色韩式；这三句不放在预制问题中。
3. 场景意图：说“给我推荐点健身好物”，观察训练、恢复、补水和数据记录商品组合。
4. 探索意图：说“有没有让我眼前一亮的好物”，观察跨品类新奇好物。
5. 也可演示出租屋、演唱会、幸福感和防晒霜四个预制 Case。
6. 每次提交后首页即刻回顶并重排；再次进入语音面板可继续补充或删除关键词。

## 三级 Intent Engine

系统先判定模式，再进入对应推荐链路：

- `product` → `constraint_ranking`：解析商品类目、属性、人群、风格、品牌和预算，进行约束筛选与排序。
- `scenario` → `scenario_bundle`：将生活任务拆成商品集合并交错展示，例如露营、搬家、养猫、装修、旅行、健身、做饭、音乐节和空间改造。
- `explore` → `inspiration_discovery`：按新鲜度、趋势性、风格、生活启发和跨品类多样性生成灵感推荐。

API 会同时返回 `mode`、`confidence`、`evidence` 与 `route`。当前离线 Demo 使用确定性规则实现这一协议，方便稳定路演；接入真实 LLM 时只需替换分类与结构化抽取实现，三条推荐链路无需改变。

商品库当前包含 196 件本地商品，支持正负向条件、连续修改、同义表达和三种意图模式间的切换。六个语音预制 Case 均至少返回 10 件带本地图片的商品；裙子连续追问保留为用户自由表达能力，不在页面上展示预制问题。

## 测试

```bash
python3 -m unittest discover tests
```

## 本地接口

- `GET /api/demo/bootstrap`：商品、初始排序和示例语句。
- `POST /api/intent/apply`：解析意图、合并会话条件并返回重排结果。
- `POST /api/intent/reset`：返回初始推荐和搜索结果。

所有状态均为页面会话级，不写入数据库。
